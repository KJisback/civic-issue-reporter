# Review Checklist

Use this after each ticket.

- Did the implementation stay within the assigned ticket?
- Did it avoid forbidden areas?
- Are all acceptance criteria satisfied?
- Were automated checks run or clearly marked not applicable?
- Is manual verification clear?
- Were dependencies avoided unless approved?
- Were docs updated where needed?
- Were privacy, security, and data risks considered?
- Are out-of-scope discoveries captured as follow-ups?
- Is the next recommended ticket still correct?
- Does `Prompt_Context_Pack.md` name the same next ticket as `Repo_Current_State.md`?
- If a UI ticket changed layout, was desktop and mobile visual verification requested?
